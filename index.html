Project concept

A high-visibility pilot infrastructure project designed around immersive technology and premium visitor experience. The pilot is structured as a scalable model: define the technical architecture, validate solutions in a real environment, then expand the ecosystem (additional venues, services, and regional connectivity) after pilot confirmation.

  • Core outcome: a working, visitable pilot attraction with measurable performance and visitor feedback.
  • Scaling model: repeatable deployment playbook for future sites and partners.
  • Partner role: technology showcase + integration collaboration + long-term deployment potential.

Why this matters (for partners)

This is a practical entry point into a new region through a flagship pilot project with a clear roadmap. It offers a real showcase environment, brand visibility, and a scalable collaboration path beyond a single installation.

  • Market entry via pilot: visible, structured, measurable deployment.
  • Showcase environment: real visitors, real usage, real data (not a “demo room”).
  • Long-term optionality: expansion phases and replication if the pilot proves out.

Status and delivery model

Delivery model: PPP / public-private cooperation framework. The public side supports land/infrastructure and regional facilitation; private side supports capital expenditure, systems, and operations. BigChanger leads concept, integration vision, and partner coordination.

  • Current stage: pre-contract / partner outreach, non-binding discussions.
  • Goal now: align on partner fit and integration roadmap for the pilot phase.

What “partner-ready” means here: clear contact, defined scope areas, and a short path to next steps (intro call → technical fit → partner program mapping → pilot architecture discussion).

  • Requested output from partners: “sounds interesting” + who we should talk to next.
  • Not requested right now: binding commitments or purchase orders.

Roadmap (pilot-first)

Clear phases: define → validate → scale.
  • Phase A: Partner alignment (now) — introductions, fit assessment, mapping partner programs and collaboration formats.
  • Phase B: Pilot architecture — high-level system design, integration approach, validation criteria, phased deployment plan.
  • Phase C: Pilot implementation — installation, commissioning, operational readiness, quality validation.
  • Phase D: Scaling — replication and expansion discussions based on pilot performance and stakeholder alignment.
